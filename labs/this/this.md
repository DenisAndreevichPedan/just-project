# Неделя 6 — заметка: `this`

Краткий конспект: чтобы после чтения можно было объяснить своими словами и пройти лабу.

---

## 1. Что такое `this` — одной фразой

`this` внутри функции — это ответ на вопрос: **«с каким объектом сейчас работаем?»**

Это не обычная переменная. Ты её не передаёшь как `title` или `count`. JavaScript подставляет `this` **в момент вызова** — и от того, **как** вызвали функцию, зависит, что там окажется.

Главное правило для обычной функции:

> **Кто вызвал — тот и задаёт `this`.**

---

## 2. Обычная функция: три простых случая

Разберём без таблиц. Смотри только на **точку перед скобками** `()`.

### Случай А: вызвали «просто функцию»

```js
function showThis() {
  console.log(this);
}

showThis();
```

Перед скобками ничего нет — только имя функции. Значит, это **не** метод объекта.

В современном Node и в модулях обычно `this` будет **`undefined`**.

Запомни: `fn()` — «голый» вызов, объекта слева нет.

### Случай Б: вызвали как метод — `объект.функция()`

```js
const ticket = {
  title: "Login bug",
  printTitle() {
    console.log(this.title);
  },
};

ticket.printTitle(); // Login bug
```

Слева от скобок стоит `ticket`. Значит **`this` = ticket**.

Метод — это та же функция, но вызванная через точку у объекта.

### Случай В: явно сказали «работай от этого объекта» — `call`

Иногда объект слева «подставляют» вручную:

```js
function showThis() {
  console.log(this.name);
}

showThis.call({ name: "Anna" }); // Anna
```

`call` значит: «вызови эту функцию и считай, что `this` — вот этот объект».

На старте достаточно знать **один** способ — `fn.call(объект)`. `apply` и `new` — позже, в лабе они не обязательны.

**Итог по пункту 2:** не учи таблицу наизусть. Смотри на вызов:

- `print()` → объекта нет → `this` не привязан к твоему ticket;
- `ticket.print()` → `this` = ticket;
- `print.call(ticket)` → ты сам привязал `this` к ticket.

## 3. Метод объекта — когда всё работает

```js
const ticketService = {
  prefix: "Ticket:",
  print(title) {
    console.log(this.prefix + title);
  },
};

ticketService.print("Broken login"); // Ticket:Broken login
```

Внутри `print` слово `this` — это **сам** `ticketService`. Через `this.prefix` метод читает поле своего объекта.

Пока вызываешь `объект.метод()` — контекст на месте.

## 4. Потеря `this` — самая частая ошибка

Метод **отвязали** от объекта и вызвали отдельно:

```js
const ticket = {
  title: "Login bug",
  printTitle() {
    console.log(this.title);
  },
};

ticket.printTitle();     // Login bug — ок

const fn = ticket.printTitle;
fn();                    // undefined — сломалось
```

Почему:

1. `ticket.printTitle()` — слева `ticket`, `this` = ticket.
2. `fn = ticket.printTitle` — ты взял **только функцию**, без объекта.
3. `fn()` — снова «голый» вызов, как в пункте 2, случай А. `this` уже не ticket.

То же самое в жизни кода:

```js
setTimeout(ticket.printTitle, 1000);  // в таймер уходит функция без ticket

const { printTitle } = ticket;
printTitle();                         // снова без ticket
```

**Как понять, что потерял `this`:** метод вынесли в переменную или отдали в колбэк, а вызывают как `fn()`, не как `obj.fn()`.

## 5. Стрелочная функция — почему у неё «нет своего `this`»

### Обычная функция

Каждый раз при вызове JavaScript **заново решает**, кто такой `this` (смотрит на `obj.fn()` или на голый `fn()`).

### Стрелочная функция `() => {}`

Она **не участвует** в этой игре. У стрелки нет своего `this`.

Она **подсматривает** `this` там, где стрелку **написали** — как будто это обычная переменная снаружи.

```js
const service = {
  label: "Tickets API",
  run() {
    const arrow = () => {
      console.log(this.label);
    };

    const regular = function () {
      console.log(this.label);
    };

    arrow();    // Tickets API
    regular();  // undefined
  },
};

service.run();
```

Что произошло:

- Стрелку создали **внутри** `run()`. В этот момент у `run` уже есть `this` = service. Стрелка это запомнила.
- `regular` — обычная функция. Её вызвали как `regular()`, без объекта слева → свой `this` = `undefined`.

**Коротко:** стрелка не спрашивает «кто меня вызвал?». Она использует `this` **родительского** кода.

### Ловушка: стрелка как метод объекта

```js
const counter = {
  count: 0,
  increment: () => {
    this.count += 1; // this — НЕ counter
  },
};
```

`increment: () =>` написан на верхнем уровне объекта. Родительский `this` там не counter, а часто `undefined`. Счётчик не растёт.

**Правило:** если методу нужен `this` самого объекта — пиши обычный метод:

```js
increment() {
  this.count += 1;
}
```

## 6. `bind` — «приклеить» объект к функции

`bind` делает новую функцию, у которой `this` **всегда** один и тот же:

```js
const print = ticket.printTitle.bind(ticket);

print();                              // Login bug
setTimeout(ticket.printTitle.bind(ticket), 0); // тоже ок
```

Без `bind` в `setTimeout` уходит «голая» функция — контекст теряется (пункт 4).  
С `bind` — как будто каждый раз вызываешь `ticket.printTitle()`, даже если передал функцию куда-то ещё.

---

## 7. Wrapper — второй способ не потерять `this`

Вместо `bind` можно обернуть вызов:

```js
function wrapPrint(ticket) {
  return function () {
    ticket.printTitle(); // вызываем через ticket — this снова ticket
  };
}

setTimeout(wrapPrint(ticket), 0);
```

Объект `ticket` лежит в замыкании. Внутри обёртки снова идёт нормальный вызов `ticket.printTitle()`.

**Когда что:**

- `bind` — короче, если нужно просто передать метод;
- wrapper — когда хочешь явно контролировать аргументы или логику до вызова.

Стрелка **внутри** метода тоже может помочь — она подхватит `this` метода:

```js
const ui = {
  label: "Create",
  startTimer() {
    setTimeout(() => {
      console.log(this.label); // this = ui
    }, 0);
  },
};
```

## 8. React — зачем это знать

В функциональных компонентах с hooks редко пишут `this.state` — состояние в `useState`.

Но идея та же: если отдаёшь «отвязанный» метод или колбэк без нужных зависимостей, он может работать со **старыми** данными (похоже на stale closure из недели 5).

В классовых компонентах раньше писали:

```js
this.handleClick = this.handleClick.bind(this);
```

чтобы при клике `this` указывал на компонент.

## Шпаргалка на одну страницу

| Ситуация | Что с `this` |
|----------|----------------|
| `fn()` | Нет объекта слева → обычно `undefined` |
| `obj.fn()` | `this` = `obj` |
| `const f = obj.fn; f()` | Потеряли `this` |
| `fn.bind(obj)` | `this` всегда `obj` |
| `() =>` внутри метода | Берёт `this` метода |
| `obj: () =>` как поле | `this` объекта **не** работает |
