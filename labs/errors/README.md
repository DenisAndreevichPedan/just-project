# Неделя 4 — Error propagation (лаба)

**Среда:** Node v22 (для запуска скриптов)  
**Тема:** как ошибки ведут себя в sync-коде, в Promise, в `async`, без обработчика и в `setTimeout`.

---

1. Прочитай заметку: [`errors.md`](./errors.md).
2. Открой `week4-error-propagation.js`, выпиши прогноз по логам для каждого кейса.
3. Запусти: `node labs/errors/week4-error-propagation.js`.
4. Сравни с фактом; в конце ответь на вопросы внизу этого README.

## 5 вопросов к AI (или к напарнику)

Попроси сгенерировать вопросы по темам ниже и ответить самому, затем попросить проверить ответы:

1. Чем отличается необработанное исключение в синхронной функции от `throw` в колбэке `.then`?
2. Почему `try { foo(); } catch` не ловит ошибку, если `foo` — `async` и внутри неё `throw`?
3. Что произойдёт с `Promise.reject(...)` без `.catch` в браузере и в Node?
4. Почему ошибка в колбэке `setTimeout` не попадает в `catch` снаружи таймера?
5. Как минимально «подписаться» на ошибку от функции, которая возвращает Promise?
